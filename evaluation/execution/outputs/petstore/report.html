Rule Sources: RFC 9110, Section 15.3.1 (200 OK) | RFC 9110, Section 15.3.5 (204 No Content) | RFC 9110, Section 9.3.5 (DELETE)

Violation #70

Route: DELETE /user/{username}

Responses: 400 Bad Request, 404 Not Found

Parameters: username (path)

Rule ID: has-422-if-params

Rule Description: Always implement a response with the status code 422 Unprocessable Content if the method contains parameters (in case of invalid semantics).

Rule Sources: RFC 9110, Section 15.5.21 (422 Unprocessable Content)
